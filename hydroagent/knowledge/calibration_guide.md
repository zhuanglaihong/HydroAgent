# HydroAgent Calibration Strategy Guide

## 率定模式选择

| 场景 | 推荐模式 | 工具 |
|------|----------|------|
| 首次率定，快速验证 | 标准 SCE-UA | `calibrate_model` |
| NSE < 0.65，疑似参数范围问题 | LLM 智能率定 | `llm_calibrate` |
| 需要多次重复评估稳定性 | 多次运行取平均 | `calibrate_model` × N |
| 多模型对比 | 分别率定后对比 | `calibrate_model` × 模型数 |

## 算法参数建议

### SCE-UA（默认推荐）

```
快速测试:  rep=200, ngs=100   （约 2-5 分钟）
标准率定:  rep=500, ngs=200   （约 10-30 分钟）
精细率定:  rep=2000, ngs=500  （约 1-2 小时）
```

- `rep`：函数评估总次数（越大越精细，但耗时）
- `ngs`：复合体数量（= 参数维度 × 2 为经验值）
- `kstop`：收敛判断步数，通常等于 rep

### GA（遗传算法）

```
标准: pop_size=40, n_generations=25
```

适合：参数维度较高（>6），搜索空间复杂。

### scipy（梯度优化）

仅适合平滑目标函数，不推荐用于水文模型率定（多模态）。

## 常见问题诊断

### NSE 持续偏低（< 0.5）

1. **数据问题**：先用 `validate_basin` 检查数据完整性
2. **参数范围太窄**：使用 `llm_calibrate`，让 LLM 分析边界效应并扩展范围
3. **模型不适用**：考虑换模型（如干旱区换用 GR4J 替代 XAJ）
4. **训练期太短**：建议至少 8-10 年

### 峰值流量系统性偏高

- GR4J：增大 x1（扩大产流水库），或增大 x4（延长汇流时间）
- 检查 PBIAS：正值表示模拟偏大

### 基流模拟差

- GR4J：检查 x2（地下水交换系数），确认范围是否包含负值
- XAJ：检查 Ds 和 Ws，控制基流启动条件

### 参数碰到搜索边界

这是 LLM 智能率定最擅长的场景：
```
用户提示: "率定结果 NSE=0.58，请用智能率定模式重新优化"
→ 系统调用 llm_calibrate，LLM 检测边界效应后扩展搜索范围
```

## 训练期/测试期建议

- **训练期**：8-10 年（包含丰、平、枯水年）
- **测试期**：5 年（不参与率定，独立验证）
- **预热期**：365 天（初始化模型状态）
- CAMELS-US 常用: 训练 1985-1995，测试 2005-2014

## LLM 智能率定的调整策略

LLM 顾问在每轮结束后分析两类问题并给出调整建议：

### 1. 参数范围调整（边界效应）
1. 参数值距边界 < 5% → 判定为"边界效应"
2. 碰到上界 → 扩展上界（×1.5 到 ×2）
3. 碰到下界 → 扩展下界（÷1.5 到 ÷2，同时保证物理合理）
4. NSE ≥ 目标值 → 无需调整，提前停止
5. 物理约束：容量参数 > 0，时基 > 0

### 2. 算法参数调整（无边界效应但 NSE 仍低）

当没有参数碰到边界但 NSE 仍低于目标时，问题往往是优化预算不足：

| 症状 | 诊断 | 建议调整 |
|------|------|----------|
| NSE < 0.4，无边界效应 | 搜索不充分 | rep 提高到 1500-2000 |
| NSE < 0.0（负值） | 算法未收敛 | rep=2000, ngs=300+ |
| 多轮结果波动大 | 随机性强 | 增大 ngs 减少随机性 |
| 干旱/半干旱流域 | 目标函数崎岖 | rep=1500, ngs=250 |

**使用建议**：对已知困难流域（半干旱、负NSE），初始调用时直接传入：
```
llm_calibrate(algorithm_params={"rep": 1500, "ngs": 300}, ...)
```
`llm_calibrate` 内部 LLM 顾问也会在发现无边界效应但 NSE 低时自动推荐更大的 rep/ngs。

## 重要：参数的归一化表示

**hydromodel 在优化过程中将所有参数归一化到 [0, 1]。**
`calibration_results.json` 中的 `best_params` 存储的是归一化值，不是物理值。

反归一化公式：
```
物理值 = lo + 归一化值 × (hi - lo)
```
其中 lo/hi 来自同目录的 `param_range.yaml`。

**诊断提示**：如果读取到的参数值全部落在 [0, 1] 区间内（例如 x1=0.97, x3=0.06），
说明读取的是归一化值而非物理值，需要先反归一化再做边界分析或物理合理性判断。

HydroAgent 的 `parse_calibration_result()` 已自动完成反归一化，正常调用无需手动处理。
若需手动验证，可读取 `param_range.yaml` 并应用上述公式。
