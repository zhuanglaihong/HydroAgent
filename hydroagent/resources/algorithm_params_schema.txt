## SCE-UA Algorithm (Shuffled Complex Evolution)

Algorithm ID: "SCE_UA" or "SCEUA" or "SCE-UA"

Parameters:
- **ngs** (int): Number of complexes (default: 300)
  - User keywords: "复合体数量", "complexes", "ngs"

- **rep** (int): Number of evolution steps (default: 1000)
  - User keywords: "迭代次数", "迭代轮数", "evolution steps", "iterations", "轮", "次"
  - IMPORTANT: This is the PRIMARY iteration parameter for SCE-UA
  - Example: "迭代500轮" → rep=500, "500 iterations" → rep=500

- **kstop** (int): Number of shuffling loops before convergence check (default: 10)
  - User keywords: "收敛检查", "convergence check"

- **pcento** (float): Percentage change for convergence (default: 0.01)
  - User keywords: "收敛阈值", "convergence threshold"

Example mappings:
- "算法迭代只需要500轮" → {"rep": 500}
- "使用SCE-UA，迭代1000次" → {"rep": 1000}
- "ngs=200, rep=800" → {"ngs": 200, "rep": 800}

## scipy Algorithm (scipy.optimize.minimize)

Algorithm ID: "scipy"

Parameters:
- **method** (str): Optimization method (default: "SLSQP")
  - User keywords: "方法", "method", "优化方法"
  - Options: "SLSQP", "L-BFGS-B", "TNC", "CG", "BFGS"
  - Example: "使用SLSQP方法" → method="SLSQP"

- **max_iterations** (int): Maximum number of iterations (default: 500)
  - User keywords: "迭代次数", "迭代轮数", "iterations", "迭代", "轮", "次"
  - IMPORTANT: This is the PRIMARY iteration parameter for scipy
  - Example: "迭代500轮" → max_iterations=500, "500次迭代" → max_iterations=500

- **ftol** (float): Function tolerance for convergence (default: 1e-6)
  - User keywords: "函数容差", "ftol", "function tolerance"
  - Example: "函数容差1e-5" → ftol=1e-5

- **gtol** (float): Gradient tolerance for convergence (default: 1e-5)
  - User keywords: "梯度容差", "gtol", "gradient tolerance"
  - Example: "梯度容差1e-4" → gtol=1e-4

Example mappings:
- "使用scipy算法，迭代500轮" → {"max_iterations": 500}
- "scipy，方法SLSQP，迭代1000" → {"method": "SLSQP", "max_iterations": 1000}
- "scipy算法，迭代300次" → {"max_iterations": 300}

## GA Algorithm (Genetic Algorithm)

Algorithm ID: "GA"

Parameters:
- **population_size** (int): Population size (default: 100)
  - User keywords: "种群大小", "population"

- **generations** (int): Number of generations (default: 1000)
  - User keywords: "代数", "迭代次数", "generations", "iterations"
  - IMPORTANT: This is the PRIMARY iteration parameter for GA

- **mutation_rate** (float): Mutation rate (default: 0.01)
  - User keywords: "变异率", "mutation rate"

- **crossover_rate** (float): Crossover rate (default: 0.8)
  - User keywords: "交叉率", "crossover rate"

Example mappings:
- "GA算法，迭代500代" → {"generations": 500}
- "种群100，迭代1000" → {"population_size": 100, "generations": 1000}

## General Rules for Parameter Extraction

**IMPORTANT: Available Algorithms**
Only the following THREE algorithms are supported:
1. **SCE_UA** (Shuffled Complex Evolution)
2. **scipy** (scipy.optimize.differential_evolution)
3. **GA** (Genetic Algorithm)

If user mentions any other algorithm name (e.g., "DE", "PSO", "DREAM"), it is NOT available.
You should either:
- Map to the closest available algorithm (DE→scipy, PSO→GA)
- Inform user that only SCE_UA, scipy, and GA are available

---

**Parameter Mapping Rules**:

1. **Algorithm-specific mapping**: Always map iteration keywords to the correct algorithm parameter
   - SCE_UA: "迭代500轮" → rep=500
   - scipy: "迭代500轮" → max_iterations=500
   - GA: "迭代500轮" → generations=500

2. **Priority**: User-specified values override defaults

3. **Output format**: Use exact parameter names from this schema
   - DO use algorithm-specific names as defined above:
     * SCE_UA: rep, ngs, kstop, pcento
     * scipy: method, max_iterations, ftol, gtol
     * GA: generations, population_size, mutation_rate, crossover_rate

4. **Validation**: If algorithm is specified, validate parameter names against this schema

5. **Default algorithm**: If no algorithm is specified, use SCE_UA as default
