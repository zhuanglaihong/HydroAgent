# HydroMCP 水文模型工具集介绍

HydroMCP是一个专业的水文模型工具集，基于Model Communication Protocol (MCP)架构，提供了完整的水文建模工作流工具。该工具集支持从数据预处理到模型率定、评估的全流程自动化处理。

## 工具概览

当前HydroMCP提供了4个核心工具，涵盖水文建模的主要环节：

1. **get_model_params** - 模型参数查询工具
2. **prepare_data** - 数据预处理工具  
3. **calibrate_model** - 模型参数率定工具
4. **evaluate_model** - 模型性能评估工具

## 详细工具说明

### 1. get_model_params - 模型参数查询工具

**功能描述：**
获取指定水文模型的详细参数信息，包括参数名称、取值范围和数量。

**支持的模型：**
- gr4j: GR4J日径流模型(4个参数)
- gr2m: GR2M月径流模型(2个参数)  
- gr3j: GR3J日径流模型(3个参数)
- gr5j: GR5J日径流模型(5个参数)
- gr6j: GR6J日径流模型(6个参数)
- xaj: 新安江模型(15个参数)
- hymod: HYMOD模型(5个参数)

**输入参数：**
- model_name (必需): 模型名称，从支持列表中选择

**输出结果：**
```json
{
  "success": true,
  "model_name": "gr4j",
  "param_names": ["x1", "x2", "x3", "x4"],
  "param_ranges": [[1, 3000], [0, 30], [1, 300], [0.5, 4]],
  "param_count": 4
}
```

**使用场景：**
- 了解模型结构和参数配置
- 率定前的参数范围查询
- 模型选择和比较分析

### 2. prepare_data - 数据预处理工具

**功能描述：**
将原始水文数据（CSV格式）转换为模型训练所需的NetCDF格式，进行数据质量检查和格式标准化。

**输入参数：**
- data_dir (必需): 数据目录路径，包含原始CSV数据文件
- target_data_scale (可选): 时间尺度，默认"D"(日尺度)
  - "D": 日尺度数据
  - "M": 月尺度数据  
  - "Y": 年尺度数据

**数据要求：**
数据目录应包含以下文件：
- basin_attributes.csv: 流域属性数据
- basin_[id]_[scale].csv: 时间序列数据（降水、蒸发、径流等）

**输出结果：**
```json
{
  "success": true,
  "message": "数据准备完成，已转换为nc格式",
  "data_dir": "/path/to/data",
  "data_scale": "D"
}
```

**生成文件：**
- attributes.nc: 流域属性数据
- timeseries.nc: 时间序列数据

**使用场景：**
- 建模前的数据准备
- 数据格式标准化
- 多流域数据整合

### 3. calibrate_model - 模型参数率定工具

**功能描述：**
使用SCE-UA优化算法自动率定水文模型参数，寻找最佳参数组合以最小化模型误差。

**输入参数：**
- model_name (必需): 模型名称
- data_dir (必需): 数据目录路径
- data_type (可选): 数据类型，默认"owndata"
- exp_name (可选): 实验名称，默认"model_calibration"
- result_dir (可选): 结果保存目录
- basin_ids (可选): 流域ID列表，默认["11532500"]
- calibrate_period (可选): 率定时间段，默认["2013-01-01", "2018-12-31"]
- test_period (可选): 测试时间段，默认["2019-01-01", "2023-12-31"]
- warmup (可选): 预热期长度(天)，默认720
- cv_fold (可选): 交叉验证折数，默认1

**算法配置：**
- 优化算法: SCE-UA (Shuffled Complex Evolution)
- 目标函数: RMSE (均方根误差)
- 最大迭代次数: 100
- 复合体数量: 30

**输出结果：**
```json
{
  "success": true,
  "message": "模型 gr4j 率定完成",
  "result_dir": "/path/to/results",
  "exp_name": "model_calibration",
  "model_name": "gr4j"
}
```

**生成文件：**
- config.yaml: 率定配置文件
- sceua_gr_model/: 率定参数目录
  - calibrated_params.csv: 最优参数
  - optimization_history.csv: 优化过程

**使用场景：**
- 新流域模型率定
- 参数敏感性分析
- 多模型比较研究

### 4. evaluate_model - 模型性能评估工具

**功能描述：**
评估已率定模型在训练期和测试期的性能，计算多种统计指标以量化模型精度。

**输入参数：**
- result_dir (必需): 率定结果目录路径
- exp_name (必需): 实验名称，应与率定时使用的名称一致
- model_name (可选): 模型名称，默认"gr4j"
- cv_fold (可选): 交叉验证折数，默认1

**评估指标：**
- R² (决定系数): 模型解释方差比例
- NSE (Nash-Sutcliffe效率系数): 模型效率评价
- RMSE (均方根误差): 绝对误差度量
- MAE (平均绝对误差): 平均绝对偏差
- PBIAS (百分比偏差): 系统性偏差评价

**输出结果：**
```json
{
  "success": true,
  "message": "模型 gr4j 评估完成",
  "model_name": "gr4j",
  "exp_name": "model_calibration",
  "metrics": {
    "训练期R2": "0.8543",
    "测试期R2": "0.7891"
  }
}
```

**生成文件：**
- basins_metrics.csv: 详细评估指标
- *_evaluation_results.nc: 模拟与观测对比数据

**性能标准：**
- R² > 0.8: 优秀模型性能
- R² > 0.6: 良好模型性能
- R² > 0.4: 可接受模型性能
- R² < 0.4: 模型性能较差

**使用场景：**
- 模型精度验证
- 不确定性分析
- 模型性能报告

## 工作流集成

### 标准建模工作流
1. **数据准备**: prepare_data → 转换原始数据为标准格式
2. **参数查询**: get_model_params → 了解模型参数配置
3. **模型率定**: calibrate_model → 自动优化模型参数
4. **性能评估**: evaluate_model → 计算模型精度指标

### 高级应用场景

**多模型比较：**
```
prepare_data → 
get_model_params(gr4j) → calibrate_model(gr4j) → evaluate_model(gr4j) →
get_model_params(xaj) → calibrate_model(xaj) → evaluate_model(xaj) →
结果比较分析
```

**交叉验证：**
```
prepare_data → 
calibrate_model(cv_fold=5) → 
evaluate_model(cv_fold=5) →
统计显著性检验
```

**参数敏感性分析：**
```
get_model_params → 
批量calibrate_model(不同参数范围) → 
批量evaluate_model → 
敏感性指标计算
```

## 常见问题与解决方案

### Q1: 数据准备失败
**症状**: prepare_data返回错误
**原因**: 数据格式不符合要求或文件缺失
**解决**: 检查数据文件完整性和格式规范

### Q2: 率定收敛失败
**症状**: calibrate_model长时间运行或参数异常
**原因**: 参数范围设置不当或数据质量问题
**解决**: 调整参数范围或增加预热期长度

### Q3: 评估指标异常
**症状**: evaluate_model返回极低或负值指标
**原因**: 模型不适用或数据时间段错误
**解决**: 检查时间段设置和模型选择

## 最佳实践建议

### 数据准备阶段
1. 确保数据完整性和一致性
2. 进行数据质量检查和清洗
3. 设置合理的时间段划分

### 模型率定阶段
1. 选择合适的率定和验证期
2. 设置充分的预热期长度
3. 根据流域特征选择模型

### 性能评估阶段
1. 使用多个评估指标
2. 进行残差分析和可视化
3. 考虑不确定性量化
