# 水文学家.skill：专家认知框架注入设计

> 版本：2026-04-14 | 定位：论文创新点 + 实现设计文档

---

## 一、问题出发点

### 1.1 现有架构能做什么

HydroAgent 已有三类知识：

| 类型 | 载体 | 内容 | 触发方式 |
|------|------|------|---------|
| 程序性 | `skills/*/skill.md` | 任务步骤：先做什么、再做什么 | 关键词匹配时注入 |
| 语义性 | `knowledge/` | 领域事实：参数范围、错误码 | Agent 主动 read_file |
| 情节性 | `memory/` + basin_profiles | 历史经验：上次NSE是多少 | search_memory 检索 |

这三类回答的是：**"什么情况下该做什么事"**。

### 1.2 专家还做了什么

对比一个水文新手和资深专家的区别，不是专家知道的事实更多，而是他们的**推理过程不同**：

```
新手：参数触边界 → 查文档 → 扩大范围 → 重新跑
专家：参数触边界 → 先问为什么 → 判断是数据问题/结构问题/范围问题 → 对症处理
```

专家拥有的东西既不是"步骤"，也不是"事实"，而是：
- **心智模型**：用什么框架看问题
- **决策启发式**：遇到X直接联想到Y（跳过显式推理）
- **反模式直觉**：一看到某些特征就知道不对劲
- **诚实边界**：知道自己的知识在哪里失效

认知科学称之为 **expert intuition**（专家直觉）——它是通过大量经验压缩成的隐式模式识别，不是可以直接言说的规则。

### 1.3 现有架构的缺口

```
当前 system prompt 层次（从高到低）：

  system.md        ← Agent 身份（我是谁、怎么工作）
  ─────────────────── ← [GAP：没有"领域专家怎么思考"] ───────
  task skill.md    ← 任务程序（这个任务的步骤）
  knowledge/       ← 语义事实（按需查阅）
  memory/          ← 情节记忆（历史经验）
```

**缺口**：没有一个层次在告诉 Agent "一个好的水文学家面对这个问题时，脑子里在想什么"。

## 二、Nuwa-Skill 的启发

`nuwa-skill`（女娲技能）项目的核心洞察：

> 专家的价值不在于他们记住了多少事实，而在于他们的**认知操作系统**——用什么框架解析问题，什么时候走捷径，什么时候警觉，以及什么时候诚实地说"我不知道"。

其蒸馏五要素：

| 要素 | 含义 | 水文领域类比 |
|------|------|-------------|
| 表达 DNA | 语气/风格 | "NSE=0.71，良好但不足以信任极端洪水预测" |
| 心智模型 | 认知框架 | 参数-过程对应；流域分类先于参数搜索 |
| 决策启发式 | 判断捷径 | 5轮无改善→先换模型，不是再加轮次 |
| 反模式 | 高频错误 | 只看NSE不看KGE；不评估测试期就宣称成功 |
| 诚实边界 | 知识边界 | XAJ对半干旱流域不适配；LLM建议参数范围只能作为先验 |

## 三、水文学家.skill 在架构中的位置

### 3.1 与五层架构的关系

```
+---------------------------------------------------------------+
|  [大脑]  agent.py (ReAct Loop)                                |
|    推理决策：下一步做什么                                     |
+-----------------------------+---------------------------------+
                              | 认知塑造（常驻注入）
+-----------------------------v---------------------------------+
|  [小脑 A]  skills/system.md                                   |
|    Agent 身份层：我是谁，我如何工作，我的原则是什么            |
+-----------------------------+---------------------------------+
                              | 领域认知塑造（常驻注入）★新增
+-----------------------------v---------------------------------+
|  [小脑 B]  expert_hydrologist/skill.md  ← 水文学家.skill      |
|    专家认知层：好的水文学家怎么思考问题（心智模型+直觉+边界） |
+-----------------------------+---------------------------------+
                              | 任务程序指引（关键词匹配注入）
+-----------------------------v---------------------------------+
|  [小脑 C]  skills/calibration/skill.md 等任务 Skill           |
|    任务程序层：这个任务的步骤、决策规则、失败恢复              |
+---------------------------------------------------------------+
```

水文学家.skill 是**小脑层的一个新子类型**，位于 system.md（Agent 身份）和 task skill.md（任务程序）之间。

### 3.2 三种 Skill 的本质区别

| 维度 | system.md（身份层） | 水文学家.skill（认知层）| task skill.md（程序层）|
|------|-------------------|----------------------|----------------------|
| 回答的问题 | 我是什么角色？ | 专家怎么思考？ | 这个任务怎么做？ |
| 注入时机 | 始终注入 | **始终注入** | 任务匹配时注入 |
| 粒度 | Agent 级别 | **领域级别** | 任务级别 |
| 内容形式 | 原则/禁令 | 心智模型/启发式 | 步骤/规则 |
| 认知科学类比 | 元认知（thinking about thinking） | 专家直觉（expert intuition） | 程序性知识（procedural knowledge） |
| 对应人类 | 职业道德/工作方式 | 多年积累的"感觉" | SOP 手册 |

### 3.3 与 knowledge/ 的区别

```
knowledge/model_parameters.md:
  "XAJ 的 K 参数代表蒸散发折算系数，典型范围 0.1-1.0"
  → 这是事实，需要时查阅

水文学家.skill 中的心智模型：
  "参数范围设定的正确顺序：先判断气候区 → 缩小合理范围 →
   让SCE-UA在物理有意义的空间内搜索；不要用默认范围蒙混"
  → 这是推理框架，始终塑造思考方式
```

Knowledge 是**字典**，水文学家.skill 是**思维方式**。

## 四、水文学家.skill 内容设计

### 4.1 心智模型（3个核心）

**模型1：参数-过程对应**
> 每个参数都对应一个物理水文过程，调参的本质是在校准"对这个流域水文过程的假设"。
> K 不合理，意味着你对蒸散发强度的假设不对，不是数字问题。

**模型2：流域分类先于参数搜索**
> 先判断流域类型（气候区 + 地貌特征），再确定参数的合理范围。
> 用默认宽范围搜索等于让优化器在物理上不可能的空间里盲目探索。

**模型3：误差分解三步**
> NSE 差时的诊断顺序：
> 1. 水量平衡（PBIAS > ±10%？）→ 蒸散发/降水数据或产流参数问题
> 2. 峰值拟合（KGE 低？）→ 汇流参数或地表径流参数问题
> 3. 基流退水（低流量期偏差？）→ 地下水参数问题

### 4.2 决策启发式（5条）

| 触发信号 | 专家直觉反应 |
|---------|------------|
| 参数触上/下边界 | 不是"扩大范围"，而是"为什么它要往那个方向走？是数据单位错了？还是模型结构不匹配？" |
| 5 轮迭代 NSE 无改善 | 不是"再加轮次"，而是"这个模型能拟合这个流域吗？" |
| 训练期 NSE 好但测试期 NSE 差 | 不是"成功"，而是"过拟合——参数数量与数据量比例失衡" |
| 相邻参数同时触边界（如 KI 和 KG 同时触上界） | KI+KG 约束：二者之和必须 < 0.7；不是分开调整，是联合约束 |
| NSE < 0 | 极端异常，先检查时间戳对齐和数据单位，不要进入参数调整 |

### 4.3 反模式（专家不会这样做）

- 不在只有训练期指标时宣称"率定成功"
- 不用单一指标（只看 NSE）评价模型好坏——至少看 NSE + KGE
- 不在 PBIAS > ±15% 时还继续关注峰值拟合——水量平衡是基础
- 不忽略边界命中就扩大范围——先理解边界命中的物理原因
- 不把模型率定结果直接外推到气候区差异显著的其他流域

### 4.4 诚实边界

- XAJ 对半干旱/超干旱流域（aridity > 2.0）结构性不适配，5 轮改善无望时直接说明
- LLM 推理出的参数范围可信度：气候区判断 > 量级判断 > 精确值；提供先验，不能替代优化
- NSE 在 [0.50, 0.65] 区间属于"可接受"而非"良好"，不应该被过度解读为成功

## 五、实现计划

### 5.1 文件结构

```
hydroagent/skills/expert_hydrologist/
├── skill.md     ← 水文学家.skill 主文件（本文档的实现）
└── __init__.py  ← 空文件（保持包结构一致）
```

### 5.2 注入方式

与 task skill.md 的**差异注入逻辑**：

```python
# agent.py _build_system_prompt() 中

# 1. 任务型 skill：关键词匹配才注入（现有逻辑）
matched_skills = self.skill_registry.match(query)

# 2. 认知型 skill：始终注入（新增）
expert_skill_path = self.skills_dir / "expert_hydrologist" / "skill.md"
if expert_skill_path.exists():
    expert_content = expert_skill_path.read_text(encoding="utf-8")
    sections.append("## Expert Hydrologist Cognitive Framework\n\n" + expert_content[:1500])
```

token 预算：约 800-1200 tokens（始终占用，需精简）

### 5.3 skill.md 的 frontmatter

```yaml
---
name: Expert Hydrologist
type: cognitive          # 新增 type 字段区分认知型 vs 任务型
inject: always           # 始终注入，不做关键词匹配
description: 资深水文学家的认知框架——心智模型、决策启发式、反模式与诚实边界
priority: 100            # 高于任务型 skill，在 system.md 之后、任务 skill 之前注入
---
```

## 六、论文创新点定位

### 6.1 现有工作的不足

当前领域 AI Agent 的领域知识注入方式：

| 方法 | 代表工作 | 局限 |
|------|---------|------|
| RAG | LlamaIndex, LangChain | 注入的是事实片段，不是推理框架 |
| Fine-tuning | 领域模型微调 | 改变权重，不可解释，不可快速迭代 |
| Prompt stuffing | 大多数 AgentIC 系统 | 注入的是规则/步骤，不是启发式 |

缺失：**专家直觉（expert heuristics）的结构化编码与注入**

### 6.2 本工作的贡献

**在三类记忆框架基础上，新增第四类：专家认知框架（Expert Cognitive Framework）**

```
三类记忆（exp1 已有）：
  程序性 → skill.md（怎么做）
  语义性 → knowledge/（查什么）
  情节性 → memory/（经历过什么）

第四类（本节新增）：
  认知框架 → 水文学家.skill（怎么想）
```

这对应认知科学中专家与新手的核心差距：
- 新手：记住规则，按步骤执行
- 专家：内化模型，模式识别，直觉判断

### 6.3 可验证的实验假设

**H1（主假设）**：注入认知框架的 Agent（K3+Cognitive）在以下方面优于仅注入程序性知识的 Agent（K2）：
- 边界命中诊断准确率更高（正确识别原因而非机械扩范围）
- 模型不适配案例的早期识别（不在无望流域浪费轮次）
- 失败原因解释的专业性（用物理语言而非通用错误描述）

**H2（对照）**：认知框架注入 vs 等量事实注入（添加更多 knowledge/ 文件）的效果对比：
- 假设：相同 token 预算下，认知框架注入效果 > 事实堆砌

**实验设计**（接入 exp4 知识消融实验）：

| 条件 | 知识配置 | 期望 NSE |
|------|---------|---------|
| K0 | 无额外知识 | 基线 |
| K2 | knowledge/ + task skill.md | 改善 |
| K3 | K2 + 水文学家.skill | 最优 |
| K3b | K2 + 等量事实（更多 knowledge/ 文件）| H2 对照 |

## 七、设计原则总结（写入论文方法论节）

> HydroAgent 的知识组织遵循认知科学的四分法，在原有三类记忆（程序性/语义性/情节性）基础上，新增**专家认知框架**层：
>
> - 程序性知识（skill.md）在任务开始时激活，指导行为决策步骤
> - 语义知识（knowledge/）作为按需参考库，Agent 遇到不确定性时主动查阅
> - 情节知识（memory/）跨会话积累经验，任务开始时摘要加载
> - **认知框架（水文学家.skill）始终激活，塑造领域内的推理模式**——不告诉 Agent 做什么，而是告诉它如何思考
>
> 这种设计的核心洞察来自认知科学对专家直觉的研究：专家与新手的差距不在于知道更多事实，而在于能够快速识别模式、调用启发式并知道自己知识的边界。通过将这种"隐性专业知识"显式编码为结构化的认知框架注入 Agent，HydroAgent 弥合了当前 LLM Agent 在领域专业推理能力上的关键缺口。
