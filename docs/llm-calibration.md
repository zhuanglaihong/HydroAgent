# LLM 智能率定

> 版本：v2.6 | 日期：2026-03-25

## 概述

HydroClaw 的 LLM 智能率定让 LLM 作为"虚拟水文学家"，指导模型参数优化过程。与传统方法的区别在于 LLM 负责**元决策**（调整参数范围、诊断问题），而 SCE-UA 负责**数值优化**（在范围内搜索最优参数）。

## 设计思路

### 为什么不让 LLM 直接调参数值？

Zhu et al. (2026) 的原始论文中，LLM 每轮直接建议参数值，通过 ~200 次迭代收敛。但在实际工程中：

| 方面 | LLM 直接调参数值 | LLM 调参数范围 + SCE-UA |
|---|---|---|
| LLM 调用次数 | ~200 次 | 2-3 次 |
| 每轮计算 | 1 次模拟 | SCE-UA 1000 轮优化 |
| API 成本 | 高 | 低 |
| 收敛可靠性 | 依赖 LLM 精度 | SCE-UA 保证全局搜索 |
| LLM 的角色 | 手动调参（不擅长） | 水文诊断（擅长） |

**结论**：让 LLM 做它擅长的事（理解物理含义、诊断问题），让算法做它擅长的事（数值搜索）。

### 工作流程

```
Round 1:
  ┌──────────────────────────────────────────────┐
  │ 默认参数范围 → SCE-UA 1000轮优化              │
  │ 结果: NSE=0.65, x1=1998 (触上界 2000)       │
  └──────────────────────────────────────────────┘
                        ↓
  ┌──────────────────────────────────────────────┐
  │ LLM 分析:                                    │
  │ "x1=1998，距上界 2000 只有 0.1%              │
  │  x1 代表产流水库容量，该流域可能需要更大容量   │
  │  建议扩展 x1 上界到 3000"                    │
  │                                              │
  │ 输出: {"x1": [1.0, 3000.0], ...}             │
  └──────────────────────────────────────────────┘
                        ↓
Round 2:
  ┌──────────────────────────────────────────────┐
  │ 新参数范围 → SCE-UA 1000轮优化                │
  │ 结果: NSE=0.74, 无参数触界                   │
  └──────────────────────────────────────────────┘
                        ↓
  ┌──────────────────────────────────────────────┐
  │ LLM 分析:                                    │
  │ "NSE=0.74 接近目标 0.75，无边界问题           │
  │  结果可以接受"                               │
  │                                              │
  │ 输出: {"no_change": true}                    │
  └──────────────────────────────────────────────┘
                        ↓
  结束，返回最优结果
```

## 使用方法

### 基本使用

```
You> 智能率定GR4J模型，流域12025000
```

### 指定目标

```
You> 智能率定GR4J模型，流域12025000，目标NSE达到0.80
```

### 指定最大轮数

```
You> 智能率定GR4J模型，流域12025000，最多3轮
```

### 直接调用工具

LLM 会自动选择 `llm_calibrate` 工具：

```python
llm_calibrate(
    basin_ids=["12025000"],
    model_name="gr4j",
    max_rounds=5,
    nse_target=0.75,
    algorithm="SCE_UA",
    algorithm_params={"rep": 1000},
)
```

## LLM 的诊断逻辑

LLM 收到的上下文包含：

1. **当前参数范围** — 每个参数的 [min, max]
2. **最优参数值** — SCE-UA 找到的最优解
3. **边界分析** — 每个参数距边界的百分比
4. **性能指标** — NSE, RMSE, KGE
5. **目标 NSE** — 用户期望的性能

LLM 据此做出判断：

| 情况 | LLM 的决策 |
|---|---|
| 参数触上界 | 扩展上界（如 x1.5 或 x2） |
| 参数触下界 | 扩展下界 |
| NSE 达标 | 停止迭代 |
| NSE 低但无触界 | 可能是模型结构问题，建议停止 |

## 支持的模型

| 模型 | 参数数量 | 默认参数范围 |
|---|---|---|
| GR4J | 4 | x1:[1,2000], x2:[-10,10], x3:[1,500], x4:[0.5,10] |
| GR5J | 5 | GR4J + x5:[0,1] |
| GR6J | 6 | GR5J + x6:[0,20] |
| XAJ | 多参 | binfilt, Dsmax, Ds, Ws, soil_d2 等 |

LLM 在每轮结束后根据 `knowledge/model_parameters.md` 中的参数物理含义做出决策，
不同模型的参数解释不同。可通过 `param_range_file` 参数传入自定义初始范围 YAML。
