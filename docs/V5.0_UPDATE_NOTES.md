# HydroAgent v5.0 更新说明

**发布日期**: 2025-12-04
**架构版本**: State Machine Architecture

---

## 🎉 重大更新

HydroAgent v5.0 引入了**状态机架构**，重新设计了系统核心，实现了更强大的错误处理、任务编排和智能决策能力。

---

## 🆕 核心特性

### 1. 状态机编排系统 (State Machine Orchestrator)

- **18 个精细状态**: 从任务识别到结果汇总，完整覆盖整个工作流
- **智能状态转换**: 基于执行结果自动选择最佳下一状态
- **自动错误恢复**: 配置失败/执行失败自动重试，最多 3 次
- **进度透明化**: 每个状态的执行情况实时追踪

```
状态机流程示例：
IDLE → RECOGNIZING_INTENT → PLANNING_TASKS → GENERATING_CONFIG
  → EXECUTING_CALIBRATION → ANALYZING_RESULTS → COMPLETED
```

### 2. GoalTracker (目标追踪器)

智能追踪 NSE 收敛情况，自动判断任务终止条件：

- ✅ **goal_achieved**: NSE 达到目标阈值（默认 0.65）
- ⏹️ **max_iterations**: 达到最大迭代次数
- 📉 **no_improvement**: 连续 5 次迭代无改善
- 📉 **performance_degrading**: 性能持续下降

**优势**: 避免无效迭代，节省计算资源

### 3. FeedbackRouter (反馈路由器)

智能错误分析与路由，自动选择最佳恢复策略：

| 错误类型 | 路由策略 | 说明 |
|---------|---------|------|
| **timeout** | retry_with_reduce_iterations | 减少算法迭代次数 |
| **configuration_error** | regenerate_config | 重新生成配置 |
| **data_not_found** | fail_with_error | 直接失败（数据问题） |
| **numerical_error** | retry_with_default_config | 使用默认参数重试 |
| **dependency_error** | fail_with_error | 直接失败（环境问题） |
| **memory_error** | reduce_complexity | 降低复杂度 |

### 4. PromptPool (提示池) with FAISS

- **语义检索**: 使用 FAISS 向量数据库，根据查询相似度检索历史成功案例
- **自动学习**: 每次成功执行自动保存提示和结果
- **动态优化**: 根据历史经验优化 Agent 提示词

**技术栈**: sentence-transformers + FAISS

### 5. Checkpoint 断点续传

- **优雅中断**: 按 Ctrl+C 自动保存当前进度
- **智能恢复**: `python run.py --resume` 从中断点继续执行
- **跨会话支持**: 支持列出历史会话并恢复

```bash
# 启动任务
python run.py "率定20个流域"

# 中断后恢复
python run.py --resume
```

## 🏗️ 架构对比

### v4.0 架构（旧）

```
IntentAgent → TaskPlanner → InterpreterAgent → RunnerAgent → DeveloperAgent
  (线性流程，错误处理简单)
```

### v5.0 架构（新）

```
Orchestrator (状态机核心)
  ├─ StateMachine (18状态管理)
  ├─ GoalTracker (NSE追踪)
  ├─ FeedbackRouter (智能路由)
  ├─ PromptPool (FAISS检索)
  └─ CheckpointManager (断点续传)

5 个 Agent (任务执行层):
  ├─ IntentAgent (意图识别)
  ├─ TaskPlanner (任务规划)
  ├─ InterpreterAgent (配置生成)
  ├─ RunnerAgent (模型执行)
  └─ DeveloperAgent (结果分析)
```

**关键改进**:
- ✅ 从**线性流程**升级为**状态机编排**
- ✅ 错误处理从**简单重试**升级为**智能路由**
- ✅ 任务管理从**一次性执行**升级为**可中断恢复**
- ✅ 提示系统从**静态模板**升级为**语义检索**

## 🎯 新增功能

### 1. 交互式命令增强

```bash
💬 HydroAgent> status      # 查看系统状态（新增）
💬 HydroAgent> history     # 查看历史会话
💬 HydroAgent> resume      # 恢复最近会话
💬 HydroAgent> examples    # 查看查询示例
💬 HydroAgent> help        # 详细帮助
```

### 2. 系统状态监控

`status` 命令实时显示：
- 当前状态机状态
- 任务进度（3/5）
- 重试计数
- Checkpoint 状态

### 3. 智能迭代终止

不再需要手动判断何时停止迭代，GoalTracker 自动：
- 检测 NSE 是否达标
- 识别无改善情况
- 避免性能下降

## 🔄 迁移指南

### 从 v4.0 升级到 v5.0

**好消息**: 用户接口**完全兼容**，无需修改查询语句！

```bash
# v4.0 和 v5.0 都支持
python run.py "率定GR4J模型，流域01013500"

# v5.0 新增功能
python run.py --resume          # 恢复会话
python run.py                   # 交互模式使用 status 命令
```

### 配置文件变化

**无变化**！`configs/definitions_private.py` 和 `configs/config.py` 格式完全相同。

### API 变化

仅内部架构变化，公共 API 保持不变：

```python
# v4.0 和 v5.0 都支持
from hydroagent import HydroAgent

agent = HydroAgent(backend="api")
agent.initialize()
agent.run_interactive()
```
