# RAG增强Agent使用示例

本文档展示如何使用集成了RAG知识库的HydroAgent进行水文建模任务。

## 🚀 快速开始

### 1. 启动RAG增强Agent

```bash
# 启动Agent（默认启用RAG）
python Agent.py

# 或者明确启用RAG
python Agent.py --enable-rag --debug
```

## 🧠 RAG增强功能详解

### 知识库自动检索

当用户提出查询时，系统会：

1. **分析用户意图**: 识别任务类型和关键实体
2. **智能检索**: 从知识库中检索相关文档片段
3. **查询扩展**: 基于领域知识扩展搜索词
4. **工作流生成**: 结合检索到的知识生成优化的执行步骤

### 支持的知识类型

- **模型文档**: GR4J、XAJ、HYMOD等模型的详细说明
- **参数配置**: 各模型参数的物理意义和推荐范围
- **率定方法**: SCE-UA、遗传算法等率定算法的使用指南
- **评估指标**: NSE、RMSE、KGE等指标的计算和解释

## 🔧 高级配置

### 自定义RAG参数

```python
# 在代码中自定义RAG配置
rag_config = {
    "knowledge_top_k": 3,        # 检索片段数量
    "knowledge_threshold": 0.4,  # 相关性阈值
    "enable_fallback": True,     # 启用回退机制
    "use_knowledge_cache": True  # 使用知识缓存
}

# 应用配置
agent.rag_workflow_manager.update_rag_config(rag_config)
```

### 禁用RAG功能

如果需要使用传统模式（不使用知识库）：

```bash
python Agent.py --disable-rag
```

## 📊 性能对比

### RAG增强 vs 基础模式

| 功能 | 基础模式 | RAG增强模式 |

| 工作流复杂度 | 基础 | 专业化 |

1. **新手用户**: 强烈推荐使用RAG增强模式

## 🐛 故障排除

### 常见问题

1. **RAG系统初始化失败**
   ```
   解决方案:
   - 检查documents目录是否存在
   - 确认hydrorag模块正确安装
   - 查看详细错误日志
   ```

2. **知识检索结果为空**
   ```
   解决方案:
   - 降低相关性阈值
   - 增加检索片段数量
   - 检查查询词是否过于具体
   ```

3. **工作流生成失败**
   ```
   解决方案:
   - 检查LLM模型是否正常
   - 尝试简化查询描述
   - 启用调试模式查看详细信息
   ```

### 性能优化

1. **启用知识缓存**: 减少重复检索开销
2. **调整检索参数**: 平衡相关性和性能
3. **使用合适的模型**: 推荐granite3-dense:8b
